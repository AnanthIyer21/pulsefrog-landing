# PulseFrog landing page

Static single page per `PulseFrog-Design-Landing-Spec.md` §5. No framework, no build step,
no external requests — three files plus the demo clip.

```
landing/
├── index.html   # content (§5.1 order: hero · hear it · benefits · waitlist · footer)
├── style.css    # §3.1 tokens, dark + light via prefers-color-scheme
├── script.js    # the Pulse animation, demo player, waitlist form
└── demo.mp3     # (added later — see below)
```

## Preview locally

```sh
cd landing
python3 -m http.server 8000
# open http://localhost:8000
```

Check both modes by toggling your OS appearance (the page follows
`prefers-color-scheme`). With "reduce motion" enabled the Pulse renders a
still frame instead of animating.

## Formspree ID (waitlist)

The form posts to a placeholder endpoint. To go live:

1. Create a form at formspree.io (free tier, notifications to ananth@sffstudio.com).
2. In `index.html`, replace `FORM_ID_TODO` in the form's `action`
   (`https://formspree.io/f/FORM_ID_TODO`) with the real ID.

Until the ID is set, submissions show a graceful mailto fallback to
`hello@pulsefrog.org` — nothing breaks.

## demo.mp3

The player loads `demo.mp3` from this directory (relative path). Until the file
exists the player shows "Demo coming very soon" and disables playback.

To add or refresh the demo, drop the clip in as `landing/demo.mp3` — no code
change needed. Per spec §5.1 it's a ~60-second real bulletin generated by the
pipeline, regenerated periodically (generation script is a separate
deliverable, ledger-logged, ~$0.05/clip).

## Deployment

**Now — GitHub Pages** (spec §5.2):

- Repo → Settings → Pages → deploy from branch, folder `/landing`
  (or a small Actions workflow that publishes `landing/` as the Pages artifact).
- Gives a free temp URL immediately; DNS stays untouched.

**Deploy day — S3 + CloudFront behind pulsefrog.org:**

- The infra spec already defines the bucket. Sync the folder:
  `aws s3 sync landing/ s3://<landing-bucket>/ --delete`
  then invalidate CloudFront (`/*`) and point DNS per the deploy-day runbook.
- The page is fully self-contained, so no CORS/CSP work is needed beyond the
  Formspree POST.
